# /script <account> — 台本ドラフト生成

## 入力
- `02_pain_points.md`（深い悩み）
- `04_matching.md`（悩み×案件の接続点・人が確定済み）
- `04a_offer_merits.md`（案件メリット）
- `03_competitor.md`（勝ち型）
- `profile.md`（勝ち型テンプレ・今の流行り形式）

## やること
1. 上記を読み込む
2. 以下の出力ルールで台本を生成し
   `accounts/<account>/logs/<今日の日付>/05_script_draft.md` に保存

## 出力ルール
1. 冒頭3秒のフックを3案（指が止まる言葉に振り切る）
2. 本編は profile.md の勝ち型の順序に厳密に従う
   （例：深い悩み指定→自分も同じだった→原因をズラす→意外な解決策→結果→軽いCTA）
3. 各シーンに撮る映像（日常/作業風景）をト書きで付ける
4. CTAは軽く・押し付けない
5. 想定尺と各シーンの秒数配分
6. 「この台本が維持率を取れる理由」を3点（検証用）

## 人の判断ポイント
工程4の接続点が確定してから実行。生成後は人が手直しして最終台本を確定。

## 禁止
実績の捏造・誇大訴求・ステマ的誘導文は生成しない。
